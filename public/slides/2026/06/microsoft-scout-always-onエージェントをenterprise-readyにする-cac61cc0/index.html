デモで終わる理由

危ない自律性

エージェントは“次に起きる問題”に先回りできますが、誤操作・情報漏えいが業務リスクになります。

事故
会議の誤取消し／連鎖送信
統制
ゼロトラスト／DLP／監査
実用

橋渡しの物語

Lobster→Scout

個人プロジェクト（例：Lobster／OpenClaw系）の面白さを、継続価値のある業務代行へ拡張し、企業向け統合へ結晶させます。

起点
OpenClawデモの“小ネタ”検証
帰結
Teams/Outlook/OneDrive/SharePoint横断

“賢さ”を統制へ

Open-Sourceを企業へ

オープンソースの“賢さ”をそのまま持ち込むのではなく、ゼロトラストとガバナンスで安全に統合します。

供給網
署名付きサプライチェーン
実行環境
ゼロトラスト前提（信頼しない）
統治
Agent 365＋Purview連携
enumeration
06 / 16

統治の要点

4つのレール

企業で止めないために、実装と運用を“統制のレール”に乗せます。

i
署名付きパッケージ導入（サプライチェーン）
ii
ゼロトラスト実行（コンテナを信頼しない）
iii
Agent 365で管理統制を一本化
iv
PurviewでM365同様のコンプライアンス／DLP信号
metaphor
07 / 16

自律性の条件

文脈と記憶

常時稼働は“質問への回答”ではなく、“状況を理解して動く”ことが価値になります。

Work IQ
理解・適応で“待たない”動作へ
Agentic Memory
覚える／忘れるを設計する
層化
無限の日記を避ける（適切性・プライバシー）
process

常時稼働の動き

“次”を扱う

単なる通知ではなく、過密検知や停滞のフォローアップ文案などを先回りで作成します。

例（予定）
過密を検知→提案
例（連絡）
意思決定が停滞→フォローアップ文案

“安全と品質”の文化

開発プロセスも統制

運用

品質

12 / 16

“方向性”の強みと限界

納得はするが証拠不足

統治の言葉が具体的で説得力がある一方、事故時の封じ込め手順や監査粒度など運用エビデンスは読み取れません。

強み
ゼロトラスト／署名付きサプライチェーン／Purview連携
不足
SLA・インシデント率・ロールバック・権限境界テスト詳細
closing

結論

Enterprise-readyの原則

Scoutは「個人の便利ツール」を「業務を前に進める基盤」へ寄せる設計原則を示し、リスクを抑えつつ可能性を残します。

原則
統制（ゼロトラスト／DLP）込みの設計
核
文脈理解（Work IQ）＋階層メモリ（忘れる）
提供
