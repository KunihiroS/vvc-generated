02 / 16
Mermaid互換・headless Rust

Merman.

Mermaid.jsの図を、ブラウザやJavaScriptなしで解析・レイアウト・描画するための、Rust実装です。
入力テキスト → semantic JSON / layout JSON → SVG、さらにPNG/JPG/PDFやASCIIへ展開できます。
PROBLEM
03 / 16
表示を作るだけでは足りない

互換性の壁

Mermaidの図生成はブラウザ依存になりがちで、サーバ/ドキュメント/CIでは再現性が崩れやすい課題があります。
Mermanは「見た目が近い」ではなく「仕様としてズレない」検証を中心に据えます。
ARCHITECTURE
04 / 16
入力から出力まで分解

パイプライン

図は段階的に分離されます：解析（semantic）→ 幾何計算（layout）→ 描画（SVG/ラスタ/テキスト）。
Stage i
semantic JSON 意味情報
Stage ii
layout JSON 配置幾何
Stage iii
SVG/PNG/PDF/ASCII 描画
ENUMERATION
05 / 16
Mermaid parityを狙う

検証ゲート

互換性（parity）は“差分検出”で担保します。golden snapshotとアップストリームSVGのDOM比較を使います。
検証 i
golden snapshot semantic/layout
検証 ii
DOMベースライン SVG

ズレ原因分解

SVG最終出力の現実

SVG内のHTMLラベル（``）が、ラスタ化（PNG/JPG/PDF）環境で正しく描けないことがあります。
問題

大きすぎる図は危険

pixmap budget

巨大図のPNG/JPG変換はメモリ使用が膨張しがちなので、既定で上限（例：1辺8192px）を設けます。
default
上限あり（例：8192px）

強みと限界（要点）

堅牢設計と注意点

強みは“検証プロセス内蔵”です。一方で、未実装図タイプや非完全パリティ領域は残り、導入時の確認が必要です。

CLOSING
13 / 16
結論

Mermanは基盤

サーバやCI、ネイティブ/ドキュメントビルドでMermaid互換を安定生成するための、出力パイプラインと互換性テスト中心の基盤です。
導入の成否は、図タイプ対応状況と、``/ラスタ化予算などの運用制約を理解できるかに依存します。
