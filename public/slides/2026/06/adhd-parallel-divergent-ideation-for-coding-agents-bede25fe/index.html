発想を、先に狭めない
ADHD 発散設計
LLMベースのコーディング・エージェントが「良さそうな案」に早く固定される問題（premature convergence）を、発想（ideation）の構造で抑える手法です。
Parallel Divergent Ideation for Coding Agents（ADHD） / 日本語概略
problem
03 / 16
早期収束が“事故”になる
Premature Convergence
「Xのやり方をいくつか」と頼むと、モデルは学習上の確率が高い“それっぽい候補”を先に出し、以後もその延長線で磨きがちです。
起きること
多様な選択肢が出ず、見落としが残る
なぜ困る
発想タスクでは「正しさ」より“見落とせない選択肢”が重要
Generator（生成）と Critic（批評）を分ける前提が役立ちます
metaphor
04 / 16
発散は隔離、評価は後
二段階 設計
発想（Diverge）は“広げるだけ”。評価（Focus / Critic）は“選別して深掘り”。同じ呼び出し内で混ぜないのが中核です。
i. Diverge
隔離されたN分岐で、多様な案を新規生成
ii. 禁止事項
generator側で評価・順位・ヘッジを禁止
iii. Focus
critic側でスコア→クラスタ→上位K案を深掘り
英語 + 日本語：diverge（発散）/ focus（焦点）/ critic（批評）
architecture
05 / 16
Diverge：N個の認知フレーム
Frame 再問い直し
同じ問題に対し、N個の“認知フレーム（regulator / speedrunner / biology / $0 budget 等）”を別々に適用し、新規呼び出しで案を作ります。
分岐の設計
分岐同士で文脈を共有しない（隔離）
generatorの制約
評価・順位付け・ヘッジ（可能性の言い換え）を禁止
狙い：最初の無難案へのアンカー（心理的固定）を回避
process
06 / 16
Focus：criticで“角度の地図”
Cluster 深掘り
生成された全アイデアを、別のcritic呼び出しでスコアリングし、クラスタリング（角度の塊を抽出）して上位K案を深掘りします。
スコアリング
各案を critic が点数化
クラスタリング
“似ている”ではなく“角度が違う”単位で整理
上位K案
リスク・最初の具体手順・子アイデアまで生成
責務の分離：広げる（Diverge）/ 実現する（Focus）
distinction
07 / 16
ToT/CoTと何が違う
同一トレース 禁止
CoT/ToTは同一トレース内で生成と探索が進みやすく、途中の出力がアンカーとして残る可能性があります。
CoT / ToTになりがち
発散・評価が混ざり、近い案が増えやすい
ADHDの差
generatorは隔離呼び出し + 評価禁止 / criticは別フェーズで実行
分岐は“次の手”ではなく“視点（vantage point）の再問い直し”
enumeration
08 / 16
分岐ドライバーは“視点の再構成”
Frame の目的
認知フレームは単なるペルソナ指定ではなく、問題の問い方（視点演算子）を変えて、表面的言い換えの増加を避けます。
regulator
監査可能性 / 記録 / 制約を重視
speedrunner
速度最優先 / 最短手順を優先
biology
生物・物理的制約の見立てを組み込む
例：$0 budget（無予算）で“実装現実”の角度を強制
evidence
09 / 16
結果：6問中5勝
5/6 勝利
6つのオープンエンドなエンジニアリング問題で、同一モデルの強い単発ベースラインと比較し、ADHDが5/6で勝利したと報告されています。
特に改善
trap detection（罠検出）
次に改善
novelty（新規性） / breadth（広さ）
位置づけ：夢を見るより“実務の事故（罠）”を減らす
distinction
10 / 16
trap detection：危険案をフラグ
事故防止 設計
“もっともらしいが現実制約で失敗する”案を critic が理由付きで罠（trap）としてフラグし、上位K深掘りの質を上げます。
trapの意味
コスト / レート制限 / 実装困難 / 精度要件などで崩れる可能性が高い案
効き方
“見栄えが良い”だけの案を混ぜにくくする
英語 + 日本語：trap（罠） / detection（検出）
comparison
11 / 16
トレードオフ：呼び出し回数が増える
10回 オーダー
ADHDは単発より多くのLLM呼び出し（概ね約10回）を要し、実時間では30〜90秒程度になり得ると述べています。
コスト要因
DivergeのN呼び出し + Focusのcritic呼び出し
向く場面
意思決定コストが高い設計型・創造型
注意
頻繁な内側ループには不向き
“費用対効果”を先に決める、という運用哲学
evidence
12 / 16
限界：judgeへの依存と検証の残課題
信頼性 の穴
評価はLLM-as-judgeに依存し、しかも同じモデル系列で判断するため、表層の好みで有利不利が出る可能性があります。
残る論点
judgeモデル依存 / 問題数6件で統計主張は小さい
対策の方向
別モデルのクロスジャッジ、必要なら人手評価・テスト計画で橋をかける
また、deepen出力形式がactionabilityに有利に働いた可能性も切り分け余地あり
closing
13 / 16
導入指針：発想領域かどうか
意思決定 に投資
ADHDは「単に良さそうな案を出す」よりも「見落としコストの高い意思決定で、発想の広がり方を固定する」設計です。
使うべき条件
正解が一つでない / 罠を踏むコストが高い / 発想フェーズで止めたい
運用のコツ
10回呼び出しのコストを承認し、必要ならクロスモデル評価やレビューで妥当性確認
英語 + 日本語：ideation（発想）/ breadth（広さ）/ trap（罠）
