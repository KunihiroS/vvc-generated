低遅延・多言語ASR
nemotron / 音声認識
Hugging Faceの NVIDIA nemotron-3.5-asr-streaming-0.6b を、運用視点で素早く理解できる要約です。
Streaming ASR（ストリーミング / ストリーム中に出力）＋単一モデルの多言語（40言語ロケール）を同時に狙います。
problem
03 / 19
問題は「遅延と効率」
遅延×精度
ストリーミングASRは、早く出すほど精度が落ちやすく、さらにチャンクごとの再計算でコストが増えがちです。
遅延（Latency）
チャンクを短くすると出力が早いが、文脈が弱くなりやすい。
遅延＝出力までの待ち時間（遅いほど待つ）
効率（Throughput）
バッファ型だと同じ音響特徴を繰り返し処理しやすい。
スループット＝並列処理できる数
architecture
04 / 19
狙いは一貫した設計
単一モデルで
即時認識
約 6億パラメータ（600M）の nvidia/nemotron-3.5-asr-streaming-0.6b は、低遅延ストリーミングと多言語を両立するための設計です。
Streaming
話している途中でも推論し続ける（低遅延）
Multilingual
単一モデルで複数言語ロケールを処理
Efficiency
キャッシュ対応で重複計算を抑える狙い
architecture
05 / 19
重複計算を減らす
Cache-Aware
FastConformer-RNNT
バッファ型で起きがちな「過去部分の再計算」を、キャッシュを保持して再利用する Cache-Aware FastConformer-RNNT が抑えると説明されています。
何が改善
ストリーミング中の非重複処理を増やし、効率を押し上げる。
同時ストリームの増加につながる可能性
何を狙う
低遅延を維持しつつ、スループット（同時処理数）を高める。
遅延だけでなく運用も最適化

08 / 19
運用の肝は「チャンク」
Latency-Accuracy調整
推論時の チャンクサイズ（例：80ms, 160ms, 320ms, 560ms, 1120ms）で、遅延と精度のバランスを用途別に切り替えられると説明されています。
短いチャンク
出力が早いが、文脈が短くなりやすい。
“待ち時間”優先
長いチャンク
文脈が増え精度面で有利になりやすいが、出力が遅れる。
“読みやすさ”優先
evidence

評価指標はWER/CER
WER / CER
品質は WER（Word Error Rate） と CER（Character Error Rate） で示され、日本語は CER が使われます（言語により評価が変わる）。
WER
単語単位の誤り率（英語などで一般的）
Word Error Rate
CER
文字単位の誤り率（日本語など表記体系で有利）
Character Error Rate

数字の注意点
再現性は現場次第
WER/CERはテキスト正規化の影響を受ける可能性があり、現場の音響（マイク・騒音・話者・言語混在率）で結果は変わります。
正規化
評価条件の差で誤り率が変わりうる
音声品質

実装条件
比較条件（GPU/設定/基準）で並列性も変わる
