VT Code

コード理解だけでなく、ファイル操作・端末（PTY）連携・ツール実行まで踏み込むコーディングエージェント。核は「シェル安全性」を多層防御で守る設計思想です。
対象：提示README断片から読み取れる範囲の整理（推測は明示的に限定）
03 / 15
problem
最大懸念

実行リスク

LLMが「正しそうに」提案しても、コマンド実行・引数注入・破壊的操作が起きれば被害が大きい。VT Codeは、そのリスクを実行側の防御で抑える方向性が強く読み取れます（promptの賢さに依存しない）。
狙い
LLMの誤りを前提に、実行経路を防御する
焦点
シェル操作（shell）とツール実行（tool）

architecture
全体像

エージェント

VT Codeは、会話だけでなく「開発プロセス」を実行対象にします。ツール・端末・ファイル操作・コンテキスト管理を束ね、外部規格にも接続しやすい基盤として設計されています。
理解
コード（context）を読み取り方針化
実行
ツール呼び出し／端末（PTY）／ファイル操作
連携
ACP / A2A / Open Responses / ATIF

enumeration
多層防御

守り

README断片からは、コマンド許可・引数検証・ワークスペース隔離・OSサンドボックス・人間承認・監査ログなど、複数の層で実行を制限する構成が見えます。
層 i
allowlist（許可リスト）＋引数バリデーション
層 ii
ワークスペース境界への隔離
層 iii
macOS Seatbelt / Linux Landlock＋seccomp
層 iv
human-in-the-loop（承認）＋ audit trail（監査ログ）

仕様

ただし提示入力はREADME中心のため、「例外時は拒否か再確認か」「許可の粒度」「ツールポリシーの既定（allow/deny/prompt）」は断定できません。安全性は“設計方針”として強調されています。
分かる

範囲
