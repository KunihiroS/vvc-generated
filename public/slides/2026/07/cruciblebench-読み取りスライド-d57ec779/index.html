02 / 18
永続世界で測る社会的行動
CrucibleBench
AIエージェントの「行動」を、永続的なテキスト世界（MUD）で50ターンにわたり観測し、NPCとの関係（信頼/疑念）を軸にスコア化するベンチマークです。
Key idea / 要点
単発の正誤ではなく、失敗が痕跡として残る環境での振る舞いを評価する
What’s special / 独自性
LLMジャッジが絡み、集計指標だけでは順位が揺れる理由を説明できない
本デッキは、提示されたPhase 1の要点と、Phase 2のベンチマーク化方針（校正・公開可能環境・プリレジストレーションの方向）を読みやすく整理します。
problem
03 / 18
順位の“見かけ”問題
集計だけでは危ない
LLMジャッジで集計した信頼性指標が同じでも、ジャッジ関連の採点要素を外すと順位が最大で6つ以上動き得ます。
Why it moves / なぜ揺れる
モデルごとの採点整合性（per-subject agreement）が揺れている可能性
Risk / 影響
利用者が結果を“確定的な序列”として扱うと解釈を誤る
論文では、アブレーション後に順位がシナリオサンプリングのノイズを超えて動いたことを、90%ペアブートストラップで報告しています。
architecture
04 / 18
永続MUDという測定器
履歴が残る
MUD（永続テキスト世界）では、同一ラン内で状態が続くため、アイテム所持や信頼状態などが後続行動に影響します。
Persistence / 永続性
一度の失敗がそのまま痕跡（テレメトリ/状態）として残る
Observed learning / 学習の観測
「学習して修正できるか」「誤作動が蓄積するか」を50ターンで見る
MUD 永続テキスト世界（Multi-User Dungeon）
Trust 信頼
Suspicion 疑念
これにより、単発プロンプト評価よりも「実運用に近い振る舞い」を測りやすくしています。
architecture
05 / 18
何を“成功”とするか
隠し目的への到達
評価は「このMUD上の社会的目的（隠し目的）」を、信頼や情報制約のある環境で達成できたかに置かれます。
State variables / 状態
NPCの trust / suspicion が会話と結果に反映される
Time horizon / 時間
50ターン単位で、関係のダイナミクスを追跡する
したがって、一般の社会的知能を直接証明するベンチマークではなく、外部への一般化には慎重さが必要です。
evidence
06 / 18
順位は“採点整合性”で揺れる
classifier-dependentの影
ジャッジ関連の採点要素（classifier-dependent dimensions）を外すと、順位が最大で6つ以上変動しました。
Observed / 観測
平均的な信頼性が高く見えても順位は動き得る
Mechanism / 機序
特定モデル・特定採点次元での整合が揺れる
Reported check / 検証
90%ペアブートストラップで“ノイズ超”を示唆
結論は「LLMジャッジ不要」ではなく、監査設計（per-subject agreement、順位安定性）が不可欠という主張です。
enumeration
07 / 18
ジャッジ不要でも見える失敗
テレメトリ由来の失敗モード
状態機械テレメトリから失敗モードを抽出でき、ジャッジに依存しない解釈可能性が示されました。
i. 対話ループ
同じNPCへ talk を繰り返すのに戦略が更新されない
ii. 誤対象への働きかけ
存在しない相手/名前を呼び続けるなど、対象がずれた行動
iii. 探索の麻痺
探索は十分にしているのに最終的に行動へ移れない
Dominant / 支配的
特に対話ループが多くのモデルで支配的だった
解釈の足場になるため、研究・改善サイクルに接続しやすい点が利点です。
distinction
08 / 18
“一般化”への線引き
証明ではなくヒント
CrucibleBenchは一般的な社会的知能の証明でも、実世界展開の結果を直接予測する最終結論でもありません。
Not / 否定
確定的なフロンティア比較ではない
But / しかし
LLMジャッジの監査不足が順位解釈を誤らせ得ることを示唆する
スコープの境界を明確にする姿勢は、科学的妥当性（妥当な主張範囲）に直結します。

09 / 18
Phase 2の狙い
ベンチマーク化を仕上げる
Phase 2は、校正（calibration）、プリレジストレーション、公開可能環境の完成に向けて開発中です。
Calibration / 校正
採点や測定の前提を安定化し、比較可能性を高める方向
Budget / 予算枠
低/基準/高の暫定予算枠が示され、パイロット後に最終配分
Phase 1時点の順位やスコアは“探索的”であるべき、という立場を保ったまま進める計画です。
metaphor
10 / 18
永続性＝“再犯”を見抜く
単発テストより実運用寄り
永続MUDでは、最初の誤りが消えないため、「やらかし→学習→修復」か「誤作動の蓄積」かが観測できます。
Static eval / 静的
1回の応答で切れる → 失敗履歴が残りにくい
Persistent eval / 永続
状態が継続 → 失敗が痕跡として追跡できる
Outcome / 価値
“修復可能性”を含む振る舞いの差が出る
これが、単なる知識・推論の強さよりも、社会的意思決定の質に近い差を出す設計意図です。
comparison
11 / 18
ジャッジの役割を切り分ける
“集計”と“監査”の違い
ジャッジ由来の採点次元を監査しないと、順位の揺れの理由を見落とす恐れがあります。
Aggregate metric / 集計指標
平均的な合意やκ（集計的一致指標）を見がち
Audit what / 監査すべき
per-subject agreement、順位安定性（アブレーション後の挙動）
CrucibleBenchの教訓は「順位を確定順として扱う前に、監査設計を同時に考えるべき」という点です。
evidence
12 / 18
対話ループが支配的
戦略更新の欠落
“失敗した方針”を変えず、同じNPCに対して一定回数以上 talk コマンドを繰り返す状態が、複数モデルで多かったとされています。
Observable symptom / 症状
永続世界で信頼や疑念が変わっても、行動方針が更新されない
Likely lesson / 示唆
“社会的最適化”というより会話戦略の適応不足の可能性
この種の失敗は、次に改善すべき点（対話方針の切替条件など）を考える足場になります。
distinction
13 / 18
実世界へは直結しない
測っているのはこのMUD上
測定対象は「隠し目的を、信頼や情報制約のあるMUD上で達成できるか」であり、現実の社会性をそのまま一般化できるわけではありません。
What it measures / 測れるもの
相互作用（会話）と状態遷移が意思決定に与える影響
What it doesn’t / 測れないもの
身体性、制度、現実の多様な文脈理解
Implication / 含意
外部妥当性（現実と相関するか）の検証が必要
利用者は「ベンチマークのスコープ」を前提として設計・解釈する必要があります。
closing
14 / 18
要点まとめ
読むべき結論は3つ
CrucibleBenchは、永続MUDで社会的行動を測りつつ、LLMジャッジの監査不足が順位解釈を揺らす点を示しています。
1. 永続性＝履歴を観測
失敗が痕跡として残り、修復可能性も見える
2. 集計だけでは危険
classifier-dependent次元の除外で順位が最大6つ以上動く
3. 解釈可能性＝失敗モード
対話ループ、誤対象、探索の麻痺をテレメトリから検出
Next / 次
Phase 2の校正と公開プロトコルで比較可能性を確立へ
「方向性は良いが、運用可能性・比較可能性はPhase 2に依存する」という読みが、現在の妥当な見立てになります。
