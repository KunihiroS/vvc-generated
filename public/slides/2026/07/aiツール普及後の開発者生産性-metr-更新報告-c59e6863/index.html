（信号が不安定）
Unreliable signal / 不安定な信号

推定の土台

問うている
核心

質問は単純に見える：「AIを使うと開発は速くなるか？」。しかしAIが普及すると、参加者の条件（どの層が参加するか）が変動し、推定が揺れる。
比較

Allowed vs prohibited / 許可/不許可

参加・タスク選択が変わる
Task selection / タスク選択

Time measurement / 時間計測
KEY DRIVERS
06 / 16
主因はこれ

歪み要因
四つ

今回の「信頼性が低い」判断は、選択効果（参加/タスク回避）に加え、報酬変更、エージェント同時稼働による時間計測の曖昧さなど複数要因の積み重ね。
i
選択効果が支配的
（AIなしを避ける）
Selection effect / 選択効果
ii
報酬引き下げで構成が変化
（150→50ドル/時）
Compensation shift / 報酬変更
iii
タスク時間が曖昧
（エージェント同時稼働）
Concurrent agents / 同時稼働
iv
タスク・参加者の取りこぼし
（AIで伸びる層が抜ける可能性）
Omitted segments / 取りこぼし

数字は弱い

薄い

推定値は「速くなる」と断言できない形。サブセットでは-18%（信頼区間 -38%〜+9%）、新規参加者では-4%（信頼区間 -15%〜+9%）。
サブセット
-18%
（-38% 〜 +9%）
Uncertain effect / 不確かな効果
新規参加者
-4%
（-15% 〜 +9%）
Low certainty / 確信が薄い
信頼区間が広く、どちら向きの結論も取りにくい。本文は「弱い証拠」かつ「信頼性が低い信号」と位置づける。

観測
下振れ方向に推定が寄る
Biased estimate / 偏った推定

INTERPRETATION
11 / 16
読み手の結論

「遅い／速い」は
保留

少なくともこの抜粋のトーンでは、「AIが遅くする」と断定できない。現時点では“確かな推定が難しい”“弱い証拠”に位置づけるべきだ。
本文の立場
推定が信頼できない
（選択効果が強い）
Trustworthiness issue / 信頼性問題
ただし
速度向上の痕跡はあり得る
（取りこぼしを疑う）
Potential upside / 上振れ可能性
結論は「効果量の断定」ではなく、「測定設計の追補が必要」という方向。
